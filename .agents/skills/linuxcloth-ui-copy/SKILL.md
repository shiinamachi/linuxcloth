# Linuxcloth UI Copy

사용자가 서비스 선택과 Windows 환경 준비를 이해하도록 문구를 검토한다. 안전·라이선스·데이터 삭제 결과는 숨기지 않되 구현 수칙과 내부 구성 요소는 기본 흐름에서 제거한다.

## 작업 순서

1. 변경 대상 XAML과 ViewModel에서 사용자에게 노출되는 문자열을 찾는다.
2. 각 문자열을 `유지`, `사용자 언어로 수정`, `기술 세부정보로 이동`, `제거`로 분류한다.
3. 기본 화면에는 사용자가 다음 행동을 결정하는 데 필요한 결과와 동작만 남긴다.
4. 진단에 필요한 원문·경로·구성 요소 이름은 `기술 세부정보` Expander 안에 보존한다.
5. 오류를 제목/설명/해결 동작으로 표현하고 원시 예외 메시지의 직접 노출을 피한다.
6. UI Headless 테스트와 금지 문구 검사를 실행한다.

## 문구 경계

- 기본 화면에서 서비스 찾기, 지원 상태, 필요한 준비, 진행 결과를 설명한다.
- 라이선스 책임, 작업 종료 시 데이터 삭제, 테스트 전용 제한은 결정 시점에 명확히 표시한다.
- QEMU, KVM, OVMF, SPICE, GuestBridge, qcow2, `/dev/kvm`, CLI, 패키지 저장소는 기본 문구에서 사용하지 않는다.
- 기술 이름이 문제 해결에 필요하면 접힌 기술 세부정보에 실제 이름과 진단 값을 함께 둔다.
- “검증 정보를 분리한다”, “패키지 이름이 아니라 능력을 확인한다” 같은 개발 방법론을 UI에 설명하지 않는다.
- 버튼은 결과를 표현한다. 예: `서비스 열기`, `Windows 환경 만들기`, `닫고 삭제`.
- 안전한 결과를 모호하게 만들지 않는다. `닫기` 대신 변경사항이 삭제되는 동작이면 `닫고 삭제`를 사용한다.

## 확인

다음을 실행하고 발견된 문자열을 의도에 따라 수정하거나 기술 영역임을 확인한다.

```bash
rg -n 'QEMU|KVM|OVMF|SPICE|GuestBridge|qcow2|/dev/kvm|CLI|기술 미리보기|패키지 이름이 아니라' src/LinuxCloth.Desktop
.agents/skills/linuxcloth-ui-verification/scripts/verify-ui.sh
```

최종 보고에 변경 전/후 핵심 문구와 사용자에게 필요한 안전 정보가 유지됐는지를 포함한다.
